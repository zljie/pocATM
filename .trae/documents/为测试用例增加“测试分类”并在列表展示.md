## 需求概述
- 在生成测试用例阶段为每条用例添加“测试分类”属性：正常路径测试、异常路径测试、边界条件测试、错误处理测试。
- 在“测试用例清单”表格中新增一列展示该分类，并在详情/编辑对话框中体现与可编辑。
- 将生成的用例（含分类）保存到列表数据源，便于统一查看与管理。

## 技术方案
### 类型扩展
- 更新 `src/types/index.ts` 的 `TestCase` 接口，新增可选字段：`category: 'normal' | 'exception' | 'boundary' | 'error_handling'`。
- 在 UI 中按中文标签渲染：
  - normal → 正常路径测试
  - exception → 异常路径测试
  - boundary → 边界条件测试
  - error_handling → 错误处理测试
- 兼容旧数据：`category` 可为空时在列表显示为“未分类”。

### 生成器修改（添加分类并可编辑）
- 文件：`src/components/TestCaseGenerator.tsx`
- 生成 mock 用例时为每条默认赋分类：
  - 正常业务流程验证 → normal
  - 数据验证测试/负向测试用例 → exception
  - 边界条件测试 → boundary
  - 异常情况处理 → error_handling
- 在“生成的测试用例”卡片中，为每条用例增加分类选择 `Select`（四个枚举值），选择结果写入 `generatedCases[i].category`。
- `onGenerate(generatedCases)` 时确保包含 `category` 字段。

### 列表展示（新增分类列）
- 文件：`src/App.tsx`
- 表头：在 `renderTableHeader()` 的 `case 'test-cases'` 中新增列“测试分类”。
- 行渲染：在 `renderTableRow()` 的 `case 'test-cases'` 中新增分类单元格，按枚举映射渲染彩色徽标（无分类显示“未分类”）。
- 详情对话框：在“测试用例详情”中新增“测试分类”信息块。
- 编辑对话框：新增分类选择 `Select`，保存时将分类写回到 `testCasesState`。

### 保存生成结果到列表
- 文件：`src/App.tsx`
- 更新 `handleTestCaseGeneration(testCases: Partial<TestCase>[])`：将生成器返回的用例合并进 `testCasesState`：
  - 补全字段：`functionId`（取自 `selectedFunctionForGeneration.functionId`）、`executionCount=0`、`status='draft'`、`createdAt/updatedAt=new Date()`
  - 保留从生成器传入的 `category`、`steps`、`expectedResult`、`priority` 等
- 关闭生成器对话框后刷新列表展示（基于现有 `testCasesState`）。

### 可选扩展（后续）
- 增加按“测试分类”的筛选（侧边或顶部 Select），默认显示全部。
- 在报表统计中增加分类维度的统计饼图或柱状图。

## 验证步骤
1. 运行 `pnpm dev`，打开清单页：生成用例并为其设置分类，点击“保存用例”。
2. 在“测试用例清单”表格中确认新增的“测试分类”列显示正确。
3. 打开“查看详情”确认分类信息；打开“编辑”对话框可修改分类并保存后列表更新。
4. 运行 `pnpm build` 确认编译通过。
